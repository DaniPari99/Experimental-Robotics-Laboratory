Assignment fsm Module¶

Clients:
‘client’:for connecting with Armor server.

The node implements the finite state machine which drives the robot through the locations of the map according to the stimulus.

class scripts.assignment_fsm.Decide(helper_interface)[source]¶

class to implement the ‘DECIDE’ state of the Finite State Machine in which the robot decides the next location to be reached.

At the beginning is done a check of the battery and if the battery is low, the robot comes back to the ‘SLEEP’ state of the fsm.

execute(userdata)[source]¶

Function executed whenever the fsm goes in the ‘Decide’ state. I control the state of the battery:

-if the battery is low: the transition returned is ‘tired’

-otherwise: the next location is chosen and the transition returned is ‘decided’

Parameters:userdata – variable passed between the states of the fsm
class scripts.assignment_fsm.Sleep(helper_interface)[source]¶

class to implement the ‘SLEEP’ state of the Finite State Machine in which the robot sleeps in location ‘E’ until the battery goes back to high state.

execute(userdata)[source]¶

Function executed whenever the fsm goes in the ‘Sleep’ state.

If the robot is not in location ‘E’ (the first time is always in E) I make the robot go back to location ‘E’ in order to recharge its battery. I control the state of the battery:

-if the battery is still low: the transition returned is ‘tired’

-otherwise: the transition returned is ‘rested’

Parameters:userdata – variable passed between the states of the fsm
class scripts.assignment_fsm.Visit(helper_interface)[source]¶

class to implement the ‘VISIT’ state of the Finite State Machine in which the robot visits the chosen location.

At the beginning is done a check of the battery and if the battery is low, the robot comes back to the ‘SLEEP’ state of the fsm.

execute(userdata)[source]¶

Function executed whenever the fsm goes in the ‘Visit’ state. I control the state of the battery:

-if the battery is low: the transition returned is ‘tired’

-otherwise: the location chosen is visited

-if the battery goes low during the visitation: the robot must go to recharge and so the returned transition is ‘tired’

-otherwise: it means that the room is visited correctly and so the returned transition is ‘visited’

Parameters:userdata – variable passed between the states of the fsm
class scripts.assignment_fsm.Wait(helper_interface)[source]¶

class to implement the ‘WAIT’ state of the Finite State Machine in which the robot waits for the ontology to be loaded.

execute(userdata)[source]¶

Function executed only at the first state (WAIT) of the finite state machine.

The returned ‘transition’ is only ‘loaded’. :param userdata: variable passed between the states of the fsm.
